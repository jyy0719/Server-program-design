먼저
플레이리스트페이지 입니다.
플레이 리스트는 사용자가 음악을 추가할 당시 자신이 원하는 음악을 자신이 만든 카테고리이름에 모아두어 재생목록을 분류할 수 있도록 만든 페이지 입니다.
먼저 이렇게 지은님이 만드신업로드 페이지에 카테고리를 추가하면 이 카테고리 별로 플레이리스트에서는 음악이 분류되어집니다. (다음)
이부분은 수업 시간에 배웠던 jstl과 sql 쿼리문을 이용해서 분리해보았습니다/
먼저 sql dao에서 디스팅트로 카테고리 이름에 중복값을 없애고 뽑아온 쿼리문값을 컨트롤러에서 받고  jsp로 보내줍니다 그다음 jsp내에서 For each문과 c:if 문을 거쳐,, 돌다가 반복되는 카테고리를 만나면 아래와같이 수행하라라는 조건을 주어  반복문과 조건문을 통해 음악카테고리를 중복되지 않게 출력할 수 있었습니다.


다음 마이페이지 입니다. 마이페이지의 경우,  저장된 회원정보를 확인 할 수 있고 이전에 회원이 업로드 했던 파일로그를 확인할수 있는 페이지 입니다.
(다음)
기존에 수업때 배웠던 멤버리스트 정보 뽑는 구조를 이용해서 dao 쿼리문으로 유저정보와 파일 업로드 정보를 가져와 화면에출력해주었습니다. 
(앞 사진으로)
그리고 여기서 만약에 유저 프로필이 null일경우에( // 보여준 다음, 다음-다음사진으로)
 기본이미지를 나타내게하기위해  jstl의 if문을 이용하여 널일 경우 기본이미지 사용하고 아닐경우 프로필 출력할 수 있도록 하였습니다.


다음 마이페이지 내에서 수정하기 버튼을 누르면 유저의 정보와 프로필 사진을 변경을 할 수있는 페이지로 넘어가서 정보를  수정할 수 있도록 만들어보았습니다.
이때는 지은님의 도움과 수업때 배웠던 멀티파츠를 이용하여 파일 업로드를 진행해보았고 회원정보를 수정할 수 있도록했습니다
먼저 프로필변경은 (다음)
jsp에서 이엔씨type을 multipart/form-data 로 돌려준 후 디스페쳐서블릿에서 겟방식으로 들어올 때랑 포스트 방식으로 들어올 때를 구분해서 
겟으로 들어왔을땐 수정페이지의 화면단을 보여주고 포스트 방식으로 들어왔을 경우 정보수정 및 파일 업로드를 할 수 있도록 했습니다.
